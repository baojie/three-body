# /serendipity — 随缘探索 Skill

## 核心理念

**从 Wiki 自身出发，借外部视角照亮盲区。**

先从现有 Wiki 随机抽取种子页面，以它们为起点去网上搜索相关分析文章，再反过来发现 Wiki 的缺口。外部分析只提供选题灵感，词条正文必须严格基于三体原文（corpus/cn/）。

---

## 参数

| 调用方式 | 说明 |
|---------|------|
| `/serendipity` | 随机从 Wiki 抽取 1–3 个种子页面 |
| `/serendipity 叶文洁` | 以"叶文洁"为灵感方向，定向探索 |
| `/serendipity 文革 红岸` | 以多个关键词为方向 |
| `/serendipity --ext` | **外部素材模式**：系统扫描原文引用的现实作者/作品，为 corpus 中有 ≥1 次命中的外部引用建页 |
| `/serendipity --ext 雨果 莎士比亚` | 以指定外部人名/作品为目标，在 `--ext` 模式下定向搜索 |

---

## 工作流程

### 步骤 1 · 抽取灵感种子

**无参数时** — 从 Wiki 随机抽取 1–3 个页面：

```bash
python3 -c "
import json, random
pages = json.load(open('wiki/public/pages.json'))['pages']
# 过滤掉章节页、列表页、分类页，只留实体/概念类
candidates = [p for p in pages if not p['id'].startswith(('三体I-','三体II-','三体III-','列表','分类'))]
seeds = random.sample(candidates, min(3, len(candidates)))
for s in seeds:
    print(s['id'], '|', s.get('type','?'), '|', s.get('description','')[:40])
"
```

**有参数时** — 将参数关键词直接作为种子方向，跳过随机抽取。

输出示例：
```
种子页面：
1. 红岸基地 | place | 叶文洁工作多年的军事天文基地…
2. 钢印族 | concept | 接受思想钢印、坚信人类必败的…
3. 云天明 | person | 程心的暗恋对象，买下一颗星星送给她…
```

读取种子页面正文，提取其中 2–3 个**尚未充分展开**的关联概念作为搜索关键词。

### 步骤 2 · 构造搜索查询

基于种子，构造 2–3 个搜索 query，从不同角度切入：

| 角度 | 构造方式 |
|------|---------|
| 人物/概念深挖 | `三体 [种子词] 分析 深度解读` |
| 比较/类比 | `三体 [种子词] [相关词] 知乎` |
| 冷门细节 | `三体 [种子词] 细节 考据 设定` |
| 学术/文化 | `刘慈欣 [种子词] 文化 哲学 解读` |

**随机性保障**：同一次 session 中，避免重复上次已探索过的 query 角度。

### 步骤 3 · 并发搜索 + 精读

同时发出 2–3 个 WebSearch 查询（并行）。

从结果中挑选 **1–2 篇最有趣或最冷门**的文章（不选最热门的），WebFetch 精读全文。

精读提取要点：
- 文章提到了哪些三体中**具体的概念、人物、场景**名称？
- 有没有**鲜少被讨论的细节**（非黑暗森林主线）？
- 有没有把种子词和**其他词条**关联起来的论述，暗示了一个潜在的缺失页面？

### 步骤 4 · 识别候选话题

从文章中提炼 3–8 个候选词条话题，格式：

```
候选话题：
1. [词条名] — 从文章哪个论点联想到 — 与种子「XX」的关联
2. ...
```

**过滤原则**：
- 外部分析的"解读框架"名（如"宇宙法家"）：若原文无此词 → 不建页
- 抽象主题（如"道德与生存"）：不建概念页，但可以检查相关页面是否需要扩写

### 步骤 5 · 对照现有 Wiki 查缺口

```bash
python3 -c "
import json
ai = json.load(open('wiki/public/pages.json'))['alias_index']
candidates = ['话题A', '话题B', '话题C']
for c in candidates:
    result = ai.get(c, 'MISSING')
    print(c, '->', result)
"
```

分类：
- `MISSING` → 真正缺口，进入步骤 6
- 已有页面但 quality=stub → 扩写候选，进入步骤 6
- 已有完整页面 → 跳过

### 步骤 6 · 原文核查（corpus_search）

```bash
python3 wiki/scripts/butler/corpus_search.py "话题名" --max 10
```

**标准模式**入选标准：
- ≥3 次命中 → 建完整页面
- 1–2 次命中 → 建 stub（仅限有独立叙事意义的词）
- 0 次命中 → 不建页（纯外部概念）

**`--ext` 模式**入选标准（外部素材）：
- ≥1 次命中 → 可建页（原文必须有明确引用，0 次仍不建）
- 命中 1 次但叙事意义重大 → 建 stub，正文以"原文引用"为核心，外部知识作补充
- 命中 1 次且仅是路过提及（无叙事作用）→ 不建页

### 步骤 7 · 撰写词条

严格依据原文撰写，遵循 CLAUDE.md 规范：

- Frontmatter 完整（id / type / label / aliases / tags / description / books / quality）
- 正文引用带段落编号：`（3-47-098）`
- wikilinks 使用 `[[词条名]]`
- 禁止将外部分析结论写入正文

**`--ext` 模式特殊规则**：外部素材页（type: person / book）允许在"背景"小节补充该作者/作品的现实信息，但必须：
1. 首段必须锚定原文引用（该处 corpus 命中段落）
2. 外部背景说明须与原文引用有直接关联（不是泛泛介绍该作者生平）
3. 质量标记为 `stub`，除非原文有多次引用且叙事分量显著

可选校验引文来源：
```bash
grep -n "关键句" "corpus/cn/三体III：死神永生.txt" | head -5
```

### 步骤 8 · 发布

```
/wiki
```

## 禁止事项

- ❌ **禁止**把外部解读框架当词条标题（除非原文有此词）
- ❌ **禁止**在词条正文引用外部分析结论
- ❌ **禁止**在未查原文的情况下直接写词条
- ❌ **禁止**建立纯粹描述现实世界视角的词条
- ❌ **禁止** git commit（由 /wiki 负责）
